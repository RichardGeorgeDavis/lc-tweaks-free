Changelog
=========

Unreleased
- Added free-edition packaging support with GitHub Releases updates, excluding pro licensing/updater code, the YITH Licence Activator, and bundled Modules features from public free builds.
- Improved the Divi Accessibility migration failure notice so hosts that cannot auto-install plugins without filesystem credentials now get an explicit manual-download fallback for the pinned `2.1.1` package instead of a generic install failure.
- Removed the standalone Divi Accessibility info card from the LC Tweaks Modules screen now that it is no longer a bundled module toggle.

1.5.1
- Removed the bundled Divi Accessibility module and refined the standalone migration so only sites that previously enabled it auto-install the pinned Divi Accessibility 2.1.1 plugin, while later removals show a recommendation instead of forcing reinstalls.
- Added a WordPress tweak to disable admin new-user registration notification emails globally while keeping user-facing registration emails intact, and made it override the WooCommerce customer-registration admin email toggle when both are enabled.

1.5.0
- Fixed the excluded YITH helper loader so it only gates on supported active YITH premium plugins without disabling the licence-preservation path entirely.
- Added lightweight request-scoped settings snapshot helpers to reduce repeated option-array lookups in admin rendering paths and reused them in key Divi/Woo/Maintenance panels.
- Hardened YITH helper loading so exclusion helpers are only included when at least one supported YITH plugin is active.
- Reduced duplicate inline asset output by deduping identical CSS/JS snippets and guarding per-context enqueue/output from running multiple times in a single request.
- Optimized settings import/update flows by skipping no-op option writes in both admin import and WP-CLI import/preset operations.

- Added a Woo Tweaks > Checkout UX toggle to rename the visible WooCommerce city field label and placeholder to "Suburb" while keeping the underlying field key unchanged.

1.4.9

- Added a Maintenance admin action to force fresh WordPress.org plugin/theme update checks on demand, with scope-aware availability based on the current user's update capabilities.
- Hardened the manual force-update-check action so it now aligns with WordPress plugin/theme update capabilities instead of relying on general settings access alone.

1.4.7.3
- Fixed the Rank Math schema preview so shared reports now use the canonical inspected URL, diff summaries refresh immediately when ignore-key filters change, and the preview falls back cleanly when diff controls are unavailable.
- Added a `Copy URL + Diff` action to the Rank Math schema preview so the inspected frontend URL and filtered diff summary can be shared together in one clipboard payload.
- Added preset filter chips for common noisy keys in the Rank Math schema preview diff, keeping them synced with the custom ignore-key field.
- Added a `Copy Diff Summary` action to the Rank Math schema preview so the filtered LC Tweaks-vs-Rank-Math change summary can be shared without copying the full JSON-LD output.
- Added a summary-only key filter to the Rank Math schema preview diff so noisy top-level fields can be ignored without affecting the raw JSON-LD preview output.
- Added an admin-only Rank Math schema preview diff that compares the final emitted JSON-LD against the Rank Math base graph for the same URL and summarizes which entities/fields LC Tweaks changed.
- Added support for additional public schema contact points in Rank Math enrichment, including channel-specific `ContactPoint` entries for items like WhatsApp, extra public emails, and sales/licensing links, plus optional `llms.txt` contact-channel output.
- Removed the duplicate LC Tweaks `Organization Description` field and now rely on Rank Math's Local SEO description as the source of truth for schema and optional `llms.txt` organization text.
